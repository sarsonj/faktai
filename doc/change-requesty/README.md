# Change Requesty

## Pravidla
1. Každý CR je samostatný soubor `CR-XXXX-popis.md`.
2. Číslování je sekvenční (`CR-0001`, `CR-0002`, ...).
3. Stav CR se řídí životním cyklem z `doc/funkcni-specifikace.md`.

## Stavový model
- `Navrženo`
- `Analyzováno`
- `Ke schválení`
- `Schváleno`
- `Implementováno`
- `Ověřeno`
- `Uzavřeno`
